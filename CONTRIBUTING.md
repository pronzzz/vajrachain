# Contributing to VajraChain

Thank you for your interest in contributing to VajraChain! We welcome contributions from the community to help improve the protocol.

## How to Contribute

1. **Fork the Repository**: Create a personal fork of the project on GitHub.
2. **Create a Branch**: Create a feature branch for your changes (`git checkout -b feature/amazing-feature`).
3. **Make Changes**: Implement your feature or fix. Ensure your code follows the project's style.
4. **Run Tests**: Verify your changes does not break existing functionality (`cargo test`).
5. **Format Code**: Run `cargo fmt` to adhere to the style guide.
6. **Commit Changes**: specific meaningful commit messages.
7. **Push to Fork**: Push your branch to your fork (`git push origin feature/amazing-feature`).
8. **Open a Pull Request**: Submit a PR to the `main` branch of the `vajrachain` repository.

## detailed Guidelines

- **Code Style**: We use standard Rust formatting. Please run `cargo fmt` before submitting.
- **Testing**: Add unit tests for new features. We aim for high test coverage.
- **Documentation**: Update documentation (GUIDE.md, comments) if you change public APIs.

## Reporting Issues

If you find a bug or have a suggestion, please open an issue on GitHub describing the problem and steps to reproduce.
